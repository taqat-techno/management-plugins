System Overviewنظرة عامة على النظام

Ragtools is the local AI-powered knowledge base system deployed on the IT PM workstation. It indexes Markdown files from 43 project folders on the D:\ drive, embedding them into a Qdrant vector database using the all-MiniLM-L6-v2 sentence-transformer model (384 dimensions, Cosine distance). The system enables semantic search across all project documentation via MCP tools integrated with Claude Code. Ragtools هو نظام قاعدة المعرفة المحلي المدعوم بالذكاء الاصطناعي المنشور على جهاز مدير مشاريع تقنية المعلومات. يقوم بفهرسة ملفات Markdown من 43 مجلد مشروع على قرص D:\.

Incidents & Lessons Learned (#363-368) الحوادث والدروس المستفادة

#363 — HIGHعالي
Coverage Audit Requires 3 Checks, Not 1تدقيق التغطية يتطلب 3 فحوصات
Answered "no new folders" but missed sub-folder drift. Rule: every audit must check (1) new top-level folders vs baseline, (2) every folder's root MD, (3) live RAG list diff.أجبت "لا مجلدات جديدة" ولكن فاتني انحراف المجلدات الفرعية.
#364 — MEDIUMمتوسط
RAG Entries Are Per Sub-Session, Not Per Root Folderإدخالات RAG لكل جلسة فرعية
Nearly added a duplicate root entry that would have bloated the index. Check existing sub-entries before adding.كادت تضاف إدخالة مكررة. يجب التحقق قبل الإضافة.
#365 — MEDIUMمتوسط
WebFetch Auto-Upgrades HTTP to HTTPSWebFetch يرقي HTTP إلى HTTPS تلقائياً
WebFetch to localhost:21420 returned ECONNREFUSED. Use curl via Bash for all localhost services.WebFetch إلى localhost أرجع خطأ. استخدم curl لجميع الخدمات المحلية.
#366 — MEDIUMمتوسط
Baselines Go Stale Within 24 Hoursالأساسات تصبح قديمة خلال 24 ساعة
Baseline from Apr 12 missed a folder created 20 hours later. Always cross-check against live sources.الأساس من 12 أبريل فاته مجلد أُنشئ بعد 20 ساعة.
#367 — HIGHعالي
Batch Scripts Skip Special Characters Silentlyالبرامج تتخطى الأحرف الخاصة بصمت
Batch created 37 files but silently skipped KhairGate (HUB). Test against edge-case names before batch runs.البرنامج أنشأ 37 ملف وتخطى KhairGate (HUB) بصمت.
#368 — LOWمنخفض
Remove-Then-Re-Add for RAG Updatesحذف ثم إعادة إضافة لتحديثات RAG
Existing entries must be deleted before re-adding (stale cache). New entries go directly via POST.يجب حذف الإدخالات الحالية قبل إعادة إضافتها.
